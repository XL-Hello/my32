# 2.8 寸 ILI9341 TFT LCD 模块

## 1. 一分钟了解

这是一个带电阻式触摸和 SD 卡槽的 2.8 寸 TFT 彩色液晶模块，核心显示控制器为 **ILI9341**。

- 原生分辨率：`240 × 320` 像素
- 常用横屏分辨率：`320 × 240` 像素
- 色彩格式：16-bit `RGB565`，约 65K 色
- 显示接口：SPI
- 附加功能：电阻式触摸、SD 卡槽
- 背光：`LED` 引脚控制，具体高低电平和是否需要限流应以模块实物为准

可以把整个模块理解成三个共享资源的外设：

```text
ESP32 SPI 总线
├── ILI9341 显示屏：SCK / MOSI / MISO / LCD_CS
├── 触摸控制器：    T_CLK / T_IN / T_DO / T_CS
└── SD 卡：          SD_SCK / SD_MOSI / SD_MISO / SD_CS
```

数据线可以共用，但每个设备必须使用独立的 `CS`，未被访问的设备必须保持片选无效。

## 2. 模块接口整理

原始资料中的 `240*RGB*320` 建议理解为 `240 × 320 RGB`，即竖屏物理分辨率为宽 240、高 320；通过 ILI9341 的方向设置可以旋转为横屏 `320 × 240`。

### 2.1 LCD 与触摸接口

| 丝印 | 含义 | 作用 |
| --- | --- | --- |
| `VCC` | 电源 | 模块电源输入，电压必须按模块实物/说明书确认 |
| `GND` | 地 | 与 ESP32 共地 |
| `CS` | LCD 片选 | 低电平选中 ILI9341 |
| `RESET` | LCD 复位 | 低电平复位，通常接 ESP32 GPIO |
| `DC` | Data/Command | 低电平发送命令，高电平发送参数或像素数据 |
| `SDI` / `MOSI` | SPI 写入数据 | ESP32 → LCD |
| `SDO` / `MISO` | SPI 读出数据 | LCD → ESP32；只写显示时可不接，但建议保留 |
| `SCK` | LCD SPI 时钟 | 由 ESP32 SPI 主机输出 |
| `LED` | 背光控制 | 可接 GPIO 或 PWM；不要直接假定为普通逻辑输入 |
| `T_CS` | 触摸片选 | 低电平选中触摸控制器 |
| `T_CLK` | 触摸时钟 | 触摸 SPI 时钟 |
| `T_DO` | 触摸数据输出 | 触摸控制器 → ESP32，通常对应 MISO |
| `T_IN` | 触摸数据输入 | ESP32 → 触摸控制器，通常对应 MOSI；丝印可能写作 `T_DIN` |
| `T_IRQ` | 触摸中断 | 触摸按下时通知 ESP32；没有使用中断时可不接 |

注意：`SDI/SDO` 是 LCD 端的命名，`MOSI/MISO` 是 SPI 总线的命名。接线时按数据方向对应：`SDI(MOSI)` 接主机 MOSI，`SDO(MISO)` 接主机 MISO。

### 2.2 SD 卡接口

右侧接口通常包括：

| 丝印 | 含义 |
| --- | --- |
| `SD_SCK` | SD 卡 SPI 时钟 |
| `SD_MOSI` | ESP32 → SD 卡 |
| `SD_MISO` | SD 卡 → ESP32 |
| `SD_CS` | SD 卡片选 |

中间的 `SD1` 是板载 SD 卡槽，不是额外的数据协议名称。LCD、触摸和 SD 卡可以共用 SPI 时钟及数据线，但 `LCD_CS`、`T_CS`、`SD_CS` 必须分别配置。

## 3. 显示工作的基本原理

ILI9341 不是把一整块屏幕当作并行 GPIO 输出，而是通过 SPI 接收“命令 + 参数 + 像素数据”。一次刷新大致经历以下过程：

1. ESP32 拉低 `RESET`，再释放复位。
2. 通过 `CS` 选中 LCD。
3. `DC=0`，发送 ILI9341 命令；`DC=1`，发送命令参数或像素数据。
4. 设置显示区域：发送列地址 `0x2A` 和页地址 `0x2B`。
5. 发送内存写命令 `0x2C`。
6. 连续发送 RGB565 像素，ILI9341 按窗口位置写入显存。
7. 拉高 `CS`，结束本次传输。

常见初始化命令的作用如下，具体参数仍应以驱动库和屏幕模组为准：

| 命令 | 作用 |
| --- | --- |
| `0x01` | 软件复位 |
| `0x11` | 退出睡眠 |
| `0x3A` | 设置像素格式；`0x55` 通常表示 16-bit/pixel |
| `0x36` | 设置行列方向、镜像和 RGB/BGR 顺序 |
| `0x29` | 打开显示 |
| `0x2A` | 设置列地址窗口 |
| `0x2B` | 设置页地址窗口 |
| `0x2C` | 开始写入显存 |

## 4. RGB565 像素格式

每个像素占 16 bit：

```text
位 15 ... 11     位 10 ... 5       位 4 ... 0
    R[4:0]           G[5:0]            B[4:0]
```

通常以大端顺序通过 SPI 发送高字节再发送低字节：

```c
uint16_t color = ((r & 0xF8) << 8) |
                 ((g & 0xFC) << 3) |
                 ((b & 0xF8) >> 3);
send_spi(color >> 8);
send_spi(color & 0xFF);
```

整屏刷新所需的数据量为：

```text
240 × 320 × 2 = 153600 字节
```

因此实际项目通常按行或按小块刷新，例如使用 DMA 缓冲区，而不是每次都在栈上创建整屏数组。

## 5. ESP-IDF 接入思路

当前仓库尚未包含 ILI9341 驱动代码；仓库中的 `app/main.c` 目前只初始化 RGB LED 和 WS2812B。后续接入时，建议按 ESP-IDF 的 `esp_lcd` 组件组织：

```text
配置背光 GPIO
    ↓
初始化 SPI bus（SCK/MOSI/MISO）
    ↓
创建 LCD SPI panel IO（DC、LCD_CS、时钟频率）
    ↓
创建 ILI9341 panel（RESET、RGB 顺序、16 bit）
    ↓
panel reset → panel init → 设置方向 → 打开显示
    ↓
panel draw bitmap（按窗口发送 RGB565）
```

ESP-IDF 自带示例 `esp-idf/examples/peripherals/lcd/spi_lcd_touch` 展示了 `esp_lcd_new_panel_io_spi()`、`esp_lcd_new_panel_ili9341()` 和触摸 SPI 设备的组合用法。若使用 LVGL，则由 LVGL 生成绘图缓冲区，再在 flush 回调中调用 `esp_lcd_panel_draw_bitmap()`。

建议的初始参数：

- SPI mode：通常为 `0`
- LCD 命令/参数位宽：`8 bit`
- LCD 像素格式：`16 bit RGB565`
- 初始 SPI 时钟：先使用 `20 MHz` 左右验证，稳定后再根据连线和模块能力提高
- LCD 与触摸：共用 SPI 数据线，使用不同 `CS`
- LVGL 缓冲区：使用 DMA 可访问内存，先按 10～20 行的小缓冲区验证

## 6. 触摸输入：HR2046

芯片丝印为 `HR20462045`。结合常见模块资料和该类芯片的命名方式，前半部分通常表示 **HR2046 触摸控制器**，末尾的 `2045` 更可能是批次、日期或封装/生产标记，不能直接当作另一个控制器型号。当前没有拿到该芯片厂商的完整原厂数据手册，因此以下内容按 HR2046 与 XPT2046/TSC2046 兼容系列的常见接口整理；真正接入前仍应以模块原理图和实测波形为准。

HR2046 用于四线电阻式触摸屏，负责给触摸面板加电、采样模拟电压并通过 SPI 输出坐标。它是**单点触摸**方案，不是电容式多点触摸控制器。

### 6.1 接口和基本参数

| 项目 | 说明 |
| --- | --- |
| 触摸类型 | 四线电阻式，单点 |
| 通信接口 | SPI，通常与 LCD 共用 SCK/MOSI/MISO |
| 片选 | `T_CS`，低电平选中触摸控制器 |
| 数据输入 | `T_IN`，对应控制器 DIN，连接 ESP32 MOSI |
| 数据输出 | `T_DO`，对应控制器 DOUT，连接 ESP32 MISO |
| 时钟 | `T_CLK`，连接 SPI SCK |
| 中断 | `T_IRQ`，通常在检测到按下时变为低电平 |
| ADC | 常见实现为 12-bit 坐标 ADC |

`T_IRQ` 只表示“可能有触摸”，不能代替坐标读取。推荐在中断触发后仍通过 SPI 读取坐标，并通过压力值或有效范围判断本次数据是否可靠。

### 6.2 SPI 读取流程

触摸部分与显示部分是两个逻辑设备，并不因为装在同一块板上就能共用 LCD 的片选。读取触摸时应：

1. 释放 `LCD_CS` 和 `SD_CS`。
2. 拉低 `T_CS`。
3. 通过 `T_CLK`、`T_IN`、`T_DO` 发送触摸控制器命令并读取 X/Y 原始坐标。
4. 根据标定值换算到 `0～239`、`0～319` 的屏幕坐标。
5. 根据 `MADCTL` 的旋转设置同步做交换轴或镜像。
6. 释放 `T_CS`。

对 HR2046 兼容协议，常见的读取顺序如下：

```text
T_CS = 0
发送 X 命令（常见命令字节 0xD0）→ 读取 12-bit X 原始值
发送 Y 命令（常见命令字节 0x90）→ 读取 12-bit Y 原始值
可选：读取 Z1/Z2，估算触摸压力
T_CS = 1
```

每次转换返回的数据通常跨越两个 SPI 字节，结果需要右移或掩码后得到 12-bit 原始 ADC 值。`0xD0`、`0x90` 是 XPT2046/HR2046 类控制器的常见命令值，但不同兼容芯片的电源模式、采样位序和命令细节可能不同，不能只凭丝印后缀保证完全一致。

一次实际读取建议至少采样 3～5 次，丢弃明显偏离值后取中值或平均值，以降低电阻屏抖动。读取期间必须保证 `LCD_CS=1`、`SD_CS=1`，否则其他 SPI 设备可能同时驱动 MISO。

### 6.3 原始坐标到屏幕坐标

HR2046 输出的是触摸面板的原始 ADC 坐标，不是直接可用于 LCD 的像素坐标。需要先校准四个角点，再做线性映射：

```text
屏幕 x = (raw_x - raw_x_min) × 239 / (raw_x_max - raw_x_min)
屏幕 y = (raw_y - raw_y_min) × 319 / (raw_y_max - raw_y_min)
```

实际方向可能出现以下情况：

- X/Y 互换：需要 `swap_xy`
- X 方向反向：需要 `mirror_x`
- Y 方向反向：需要 `mirror_y`
- 横屏显示：先根据 LCD 的 `MADCTL` 方向旋转，再同步变换触摸坐标

校准时建议在屏幕左上、右上、左下、右下显示目标点，记录每个目标点对应的原始 `raw_x/raw_y`，不要直接套用其他模块的固定数值。不同触摸面板、装配方向和屏幕旋转都会导致校准参数不同。

### 6.4 ESP-IDF 接入建议

当前仓库自带的 ESP-IDF 示例包含 SPI LCD 和 `STMPE610` 触摸控制器示例，但没有看到 HR2046 的现成驱动。对本模块可以采用两种方式：

1. 直接使用支持 XPT2046/HR2046 兼容协议的第三方组件，并核对其 SPI 模式、命令字节和坐标位序。
2. 基于 `spi_device_polling_transmit()` 或 `spi_device_transmit()` 编写一个小型 HR2046 读取组件，再将读取结果接到 LVGL 的输入回调。

驱动层建议拆成以下接口，方便替换和调试：

```c
esp_err_t hr2046_init(const hr2046_config_t *config);
esp_err_t hr2046_read_raw(uint16_t *raw_x, uint16_t *raw_y, bool *pressed);
void hr2046_calibrate(const hr2046_calibration_t *calibration);
```

如果暂时不使用 `T_IRQ`，可以将其配置为 `GPIO_NUM_NC`，按固定周期轮询；如果使用中断，ISR 中只记录事件或发送任务通知，不要在 ISR 内执行 SPI 事务。

仅凭丝印 `HR20462045` 仍不能确认供电电压、SPI mode、`T_IRQ` 的电气细节和压力命令是否完全兼容 XPT2046。首次调试应使用逻辑分析仪确认：`T_CS` 是否独立拉低、MOSI 是否发送命令、MISO 是否有返回，以及返回数据是否随手指位置变化。

## 7. 背光、供电和接线注意事项

- 先确认模块 `VCC` 的电压要求，以及板上是否已有 3.3 V 稳压和电平转换。
- ESP32 与模块必须共地。
- `LED` 可能是背光阳极/控制输入，不能仅凭名称判断 GPIO 的高低有效；首次上电建议串联限流或使用合适的背光驱动电路，并查实物资料。
- LCD、触摸和 SD 卡的 SPI 设备都要有独立 `CS`，避免多个设备同时响应。
- SPI 线较长或频率较高时容易出现花屏、读写失败，可先降低频率并缩短连线。
- `MISO` 总线上的设备在未选中时必须释放总线；若某个模块没有正确三态输出，会影响其他设备通信。
- 横竖屏方向、RGB/BGR 顺序、颜色字节序不匹配时，常见现象是画面旋转错误、颜色偏蓝/偏红或图像镜像。

## 8. 建议的验证顺序

不要一开始同时启用 LCD、触摸和 SD 卡，建议逐层验证：

1. 只接 LCD：复位、初始化，显示纯色、红绿蓝三色和边框。
2. 验证窗口刷新：在指定坐标绘制矩形，确认 X/Y 方向和颜色顺序。
3. 打开背光并确认亮度控制正常。
4. 加入触摸：打印原始 X/Y、换算后的坐标和按下/释放状态。
5. 最后加入 SD 卡：读取文件或写入测试文件，确认共享 SPI 时片选管理正确。

建议保留以下日志，便于区分“屏幕未初始化”和“像素数据错误”：

```text
LCD SPI bus initialized
ILI9341 reset/init completed
LCD orientation: portrait/landscape
LCD test color flushed: R/G/B
Touch controller initialized
SD card mounted
```
