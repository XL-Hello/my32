# 实施设计方案（待审核，不含代码）

## 1. 目标与实施边界

本方案在现有 ESP-IDF + LVGL 工程内实现上述 240×320 RGB565 LCD UI Demo。所有新增 UI 源码、资源声明和 UI 配置均放在 `app/ui/` 下；`app/main.c`、`app/display/` 只在接入阶段按最小范围修改调用点或构建清单，不承载 UI 业务逻辑。

本期交付范围为：首页、相册占位页、控制中心、设置页，以及文档定义的手势和控件交互。相册图片解码、设置二级页面、真实 Wi-Fi/蓝牙/背光驱动和持久化不在本期实现；这些功能将通过服务接口预留，Demo 先使用内存状态和占位图展示。

## 2. 技术决策

- UI 框架：沿用工程已集成的 LVGL，所有 LVGL 对象创建、销毁及状态更新均在现有 LVGL 任务上下文中执行。
- 尺寸：定义统一的 `UI_SCREEN_WIDTH=240`、`UI_SCREEN_HEIGHT=320`、`UI_SAFE_MARGIN=10`，禁止业务页面散落屏幕尺寸字面量；布局优先使用 Flex/Grid 和上述尺寸常量。
- 页面策略：首页和相册页均只创建一次，由页面控制器显示/隐藏；控制中心和设置页同样常驻创建，通过显示、隐藏和层级管理切换，避免反复分配造成碎片。
- 图标：本期采用 LVGL 内置符号或位图资源适配层，业务代码只引用语义化图标标识。后续替换为 24×24 单色图标时不影响页面代码。
- 动画：控制中心使用 LVGL 位移动画从屏幕顶部滑入/滑出；左右页面切换首版采用无动画显示切换，后续可在页面控制器中替换为滑动动画。
- 状态：亮度、音量和 4 个开关统一保存于 UI 状态模型；控制中心、首页状态行和设置列表通过状态变更通知同步刷新，避免组件间直接互相修改。

## 3. 推荐目录与职责

```text
app/ui/
├── demo.md                         # 需求与本设计方案
├── ui.h / ui.c                     # UI 对外唯一入口：初始化、定时刷新、销毁（如需）
├── ui_config.h                     # 分辨率、间距、颜色、动画时长、触控阈值等全局常量
├── ui_types.h                      # 页面、覆盖层、设置项、图标等公共枚举与数据类型
├── ui_state.h / ui_state.c         # UI 状态模型、读写接口与订阅/刷新通知
├── ui_controller.h / ui_controller.c # 页面/浮层状态机与所有交互动作入口
├── ui_style.h / ui_style.c          # 颜色、字体、通用 LVGL 样式初始化和释放
├── ui_assets.h / ui_assets.c        # 图标、相册占位图等资源的统一映射
├── ui_gesture.h / ui_gesture.c      # 触摸起止坐标判定，转换为语义化手势事件
├── pages/
│   ├── home_page.h / home_page.c    # 首页创建、显示和局部刷新
│   ├── album_page.h / album_page.c  # 相册创建、显示和局部刷新
│   └── settings_page.h / settings_page.c # 设置全屏浮层创建与状态刷新
├── overlays/
│   └── control_center.h / control_center.c # 控制中心创建、动画与控件刷新
└── widgets/
    ├── ui_slider_row.h / ui_slider_row.c  # “标题 + 0~100 滑块”复用组件
    ├── ui_toggle_row.h / ui_toggle_row.c  # 快捷开关卡片复用组件
    └── ui_setting_item.h / ui_setting_item.c # 设置列表项复用组件
```

约束：`pages/` 和 `overlays/` 只负责视图构建、事件转发与刷新，不直接保存全局业务状态；`widgets/` 不引用任何具体页面；只有 `ui_controller` 决定显示哪个页面、哪个浮层及动画。这样新增第三个页面、一个快捷开关或一个设置项时只需新增对应模块或配置表。

## 4. 初始化与数据流

```text
lvgl_port 的 LVGL 任务
        │
        └── ui_init()
              ├── ui_style_init() / ui_assets_init()
              ├── ui_state_init()：装载 Demo 默认值
              ├── 创建首页、相册、控制中心、设置页
              ├── 注册全屏手势监听
              └── ui_controller_show_home()

用户事件 → 页面/组件事件回调 → ui_controller
          → ui_state_set_xxx() → 通知各视图 refresh()
          → 页面显示/隐藏或浮层动画
```

`ui.c` 的初始化函数是 UI 的唯一入口，实施时由 `lvgl_port.c` 中当前调用 `lvgl_hello_world_create()` 的位置替换调用。此替换仅为接入，不把 UI 实现移出 `app/ui/`。

## 5. 状态模型与默认值

| 字段 | 类型/范围 | 默认值 | 消费位置 |
| --- | --- | --- | --- |
| `active_page` | `HOME` / `ALBUM` | `HOME` | 页面控制器 |
| `overlay` | `NONE` / `CONTROL_CENTER` / `SETTINGS` | `NONE` | 页面控制器 |
| `brightness` | 0~100 | 70 | 控制中心、设置页 |
| `volume` | 0~100 | 60 | 控制中心、设置页 |
| `wifi_enabled` | 布尔 | `true` | 控制中心、首页、设置页 |
| `bluetooth_enabled` | 布尔 | `false` | 控制中心、首页、设置页 |
| `screen_always_on` | 布尔 | `false` | 控制中心 |
| `silent_mode` | 布尔 | `false` | 控制中心 |
| `battery_percent` | 0~100 | 78 | 首页 |
| `current_photo` | 非负索引 | 0 | 相册页 |
| `album_playing` | 布尔 | `false` | 相册页 |

状态模型只保存 UI 所需的快照。后续接入真实硬件时，由适配服务调用同一组 `ui_state_set_*` 接口，或由 UI 控制器再调用驱动适配接口；页面代码无需改动。

## 6. 页面、浮层与事件设计

### 6.1 层级

根屏幕下按固定顺序创建三个容器：页面层、控制中心层、设置页层。后创建的设置页天然在最上层；控制中心显示时位于页面层上方；设置页显示时覆盖全部内容。每个容器宽高均为 240×320，默认禁用超出屏幕的滚动。

### 6.2 页面控制状态机

```text
HOME ──左滑──> ALBUM
ALBUM ──右滑──> HOME

HOME / ALBUM ──顶部向下滑──> CONTROL_CENTER（保留来源页面）
CONTROL_CENTER ──向上滑──> 返回来源页面
CONTROL_CENTER ──设置按钮──> SETTINGS
SETTINGS ──返回按钮──> CONTROL_CENTER
```

补充规则：控制中心已打开时，页面左右滑动不生效；设置页全屏显示时只接受返回按钮，不触发下拉控制中心。`active_page` 与 `overlay` 分开保存，故关闭浮层时可准确恢复首页或相册。

### 6.3 手势仲裁

在页面层根容器注册触摸事件，记录按下和释放坐标：

- 位移绝对值小于触控阈值（建议 20px）按点击处理，不触发页面切换。
- 起点在顶部 40px 内且纵向位移 ≥ 阈值、方向向下：打开控制中心。
- 控制中心可见且纵向位移 ≥ 阈值、方向向上：关闭控制中心。
- 无浮层时，横向位移 ≥ 阈值且横向大于纵向：按当前页面切换首页/相册。
- 控件自身（滑块、开关、按钮）应停止事件冒泡或被手势仲裁识别为控件操作，防止拖动滑块误判为页面滑动。

### 6.4 各视图实现要点

- 首页：顶部 0~40px 留空；时钟区域使用定时器每分钟刷新，日期和星期由时间服务适配层提供；状态行由状态模型拼装；底部提示固定在 260~320px。
- 相册：标题栏及返回提示、220×215 预览画布、三等分底栏均由独立容器组成。上一张/下一张只改变 `current_photo`；播放键切换 `album_playing`，自动轮播定时器留作后续实现点。
- 控制中心：面板自身高度 220px，初始位于 `y=-220`；显示动画移动至 `y=0`，隐藏动画移动回 `y=-220` 后设为隐藏。亮度与音量复用滑块行，四个快捷开关由配置数组生成两行两列布局。
- 设置页：全屏白色容器，顶部固定导航栏；设置项由静态描述数组驱动（图标、标题、状态格式化函数、是否预留），通过复用列表项生成。点击列表项只保留事件入口，不创建二级页面。

## 7. 字体、中文与资源方案

当前 `lv_conf.h` 仅启用了 14px Montserrat，无法完整展示需求中的 10/12/16/32px 及中文。实施前应确认目标固件的字库方案：

1. 数字时钟使用 LVGL 内置的 `Montserrat 32`（开启对应配置），可保证 `HH:MM` 等宽和清晰。
2. 中文文本使用从开源中文点阵字库转换生成的 10/12/16px LVGL 字体，仅收录本 Demo 使用的汉字、数字与符号，避免全量中文字库占用 Flash。
3. `ui_style` 提供 `clock`、`title`、`body`、`hint` 四种语义化字体访问，不允许页面直接依赖具体字库符号。
4. 图标统一为 24×24、单色、RGB565/带透明掩码的资源；相册占位图首版以 LVGL 容器、边框和“图片预览”文字绘制，无需图片解码器。

若评估后固件 Flash 或 RAM 不适合加入中文字体，应在开发前确认允许使用可显示中文的现有字库或将演示文字替换为英文；不建议让中文显示退化为方块字符。

## 8. 触控、性能与适配要求

- 视觉控件可小于 40×40px，但所有可点击对象的实际触控容器最小为 40×40px。
- 颜色统一由 `ui_config.h` 定义为 RGB888 语义色并在 LVGL 创建时转换；显示端维持 RGB565 输出。
- 不在手势回调中做阻塞操作；仅更新状态和发起 LVGL 动画。
- 页面和浮层对象一次创建后复用；不在时钟刷新、滑块拖动等高频路径动态创建字体、图片或大量对象。
- 双缓冲策略由现有显示端口负责。UI 侧避免整屏强制重绘，仅失效发生变化的标签、滑块或开关。

## 9. 分阶段实施与验收

| 阶段 | 内容 | 验收标准 |
| --- | --- | --- |
| 1 | 新建 UI 基础层、配置、状态、样式和唯一入口；接入 LVGL 启动流程 | 上电默认显示 240×320 首页，无 `Hello world` 残留 |
| 2 | 完成首页与相册页及左右滑动 | 首页左滑到相册、相册右滑返回；点击/短位移不误切页 |
| 3 | 完成控制中心、滑块、四个开关与上下滑 | 任意页面顶部下滑打开，控件状态可同步，向上滑关闭并返回原页 |
| 4 | 完成全屏设置页、返回链路与状态展示 | 设置页从控制中心打开，返回后仍显示控制中心，数值/开关状态一致 |
| 5 | 接入中文点阵字体、图标资源，进行目标板性能调优 | 中文无乱码；交互无明显闪烁；`idf.py build` 通过 |

建议在阶段 2、4、5 分别增加可观察的串口日志（页面切换、浮层状态、关键状态值），并在目标板上手工验证触摸路径。若后续项目已有自动化触摸注入能力，再补充 UI 事件级测试。

## 10. 待确认事项

- 当前实物触摸驱动的坐标方向、有效区域和 LVGL 输入设备注册方式尚未在 UI 层看到；实施前需以现有驱动行为校准手势方向与阈值。
- 需求描述“相册 APP（首页向右滑动切换）”与前文及流转图“首页左滑进入相册”冲突。本方案以流转图和触控规则为准：首页左滑进入相册，相册右滑返回首页。
- 需求的“0~100 档位”在 UI 上按连续的 0~100 整数滑块实现；亮度、音量是否需要立即下发至真实硬件，待后续硬件接口确认后再接入。
